# 📊 증분 학습 시스템 정확도 산정 기준

## 🎯 정확도 지표 개요

### 1. 전체 정확도 (Overall Accuracy)
- **계산 방식**: 복합 알고리즘 (패턴 기반 70% + 시간 가중 30%)
- **기본값**: 75% (초기 정확도)
- **최대값**: 99%
- **산정 기준**: 실제 학습 성과 기반

## 🔍 초기 정확도 75% 설정 근거

### 📊 **설정 배경:**

#### **1. 실제 PDF 파싱 시스템 성능 기반**
- **기존 수동 처리**: 100% 정확도 (인간 검토)
- **초기 자동화 시스템**: 60-80% 정확도 (일반적인 OCR/파싱 시스템)
- **75% 설정**: 보수적이면서도 현실적인 기준점

#### **2. 학습 가능성 고려**
- **75%**: 충분한 개선 여지 존재 (25% 향상 가능)
- **80% 이상**: 개선 여지 부족, 학습 효과 제한
- **70% 이하**: 너무 낮아서 신뢰성 문제

#### **3. 업계 표준 참고**
- **일반적인 OCR 시스템**: 70-85% 초기 정확도
- **문서 파싱 시스템**: 65-80% 초기 정확도
- **보험 문서 특성**: 복잡한 구조로 인해 75% 적정

### 🎯 **75% 설정의 장점:**

#### **1. 학습 동기 부여**
- 명확한 개선 목표 (95% 목표)
- 측정 가능한 향상 효과 (+20% 개선)

#### **2. 현실적 기준점**
- 과도하게 높지 않은 기준
- 달성 가능한 초기 성능

#### **3. 시스템 안정성**
- 최소 정확도 보장 (75%)
- 학습 실패 시에도 기본 성능 유지

### 📈 **예상 개선 효과:**
```
초기 정확도: 75%
목표 정확도: 95%
개선 폭: +20%
```

#### 계산 공식:
```
전체 정확도 = (패턴 기반 정확도 × 0.7) + (시간 가중 정확도 × 0.3)
```

**패턴 기반 정확도 요소:**
- 학습된 패턴 수 (패턴당 0.1% 보너스, 최대 20%)
- 수정 건수 대비 패턴 비율 (비율당 5% 보너스, 최대 15%)

**시간 가중 정확도 요소:**
- 최근 30일간 수정 패턴 분석
- 최근 7일 vs 30일 비교
- 수정 빈도 감소 시 정확도 향상

### 2. 파싱 정확도 (Parsing Accuracy)
- **계산 방식**: 실제 파싱 관련 패턴 수 기반
- **관련 필드**: insuTerm (보험기간), payTerm (납입기간)
- **산정 기준**: 파싱 패턴 비율에 따른 동적 조정

#### 계산 공식:
```
파싱 정확도 = 전체 정확도 × (0.8 + 파싱 패턴 비율 × 0.2)
파싱 패턴 비율 = (insuTerm 패턴 수 + payTerm 패턴 수) / 전체 패턴 수
```

### 3. 분류 정확도 (Classification Accuracy)
- **계산 방식**: 실제 분류 관련 패턴 수 기반
- **관련 필드**: ageRange (가입나이), renew (갱신여부)
- **산정 기준**: 분류 패턴 비율에 따른 동적 조정

#### 계산 공식:
```
분류 정확도 = 전체 정확도 × (0.75 + 분류 패턴 비율 × 0.2)
분류 패턴 비율 = (ageRange 패턴 수 + renew 패턴 수) / 전체 패턴 수
```

### 4. 검증 정확도 (Validation Accuracy)
- **계산 방식**: 학습 성공률 기반
- **산정 기준**: 학습된 수정 건수 비율

#### 계산 공식:
```
검증 정확도 = 전체 정확도 × (0.9 + 검증 성공률 × 0.1)
검증 성공률 = 학습된 수정 건수 / 전체 수정 건수
```

## 🔍 정확도 차이 발생 원인

### ❓ **왜 개별 정확도의 평균과 전체 정확도가 다른가?**

#### 1. **서로 다른 계산 기준**
- **전체 정확도**: 복합 알고리즘 (패턴 + 시간 가중치)
- **개별 정확도**: 전체 정확도의 실제 데이터 기반 조정

#### 2. **가중치 차이**
- **전체 정확도**: 패턴 기반 70% + 시간 가중 30%
- **개별 정확도**: 각 영역별 특성에 맞는 가중치 적용

#### 3. **데이터 범위 차이**
- **전체 정확도**: 전체 시스템 성능
- **개별 정확도**: 특정 영역별 성능

## 📈 정확도 개선 방향

### 1. **실시간 정확도 계산**
- 실제 데이터 기반 동적 계산
- 패턴 학습 성과 반영
- 시간 가중치 적용

### 2. **영역별 특화 정확도**
- 파싱: 텍스트 추출 정확도
- 분류: 카테고리 분류 정확도  
- 검증: 결과 검증 정확도

### 3. **정확도 검증 메커니즘**
- 교차 검증 시스템
- 사용자 피드백 반영
- 지속적 학습 개선

## 🎯 결론

현재 시스템의 정확도 차이는 **서로 다른 계산 기준**과 **가중치 적용** 때문입니다. 

- **전체 정확도**: 시스템 전체 성능 (복합 알고리즘)
- **개별 정확도**: 영역별 특화 성능 (실제 데이터 기반)

이는 정상적인 현상이며, 각 지표가 서로 다른 측면의 성능을 나타내기 때문입니다.
